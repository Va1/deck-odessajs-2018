Theses

  • imperative is fine for small things
  • complexity grows super fast with size
  • complex async flow is painful
  • impure code makes testing and debugging painful (Angular 1 DI helps with testing)

Reactive, Stateful

(2013-2015)

Example: React 

        render () {
          const { fetchItems } = this
          const { items, error } = this.state
         
          return (
            <div>
              <button onClick={fetchItems}>
                Fetch Items
              </button>
              { error === null
                ? <ItemsList items={items} />
                : <Error error={error} /> }
            </div>
          )
        }
      
        state = { items: [], error: null }
         
        fetchItems () {
          $.get('/items')
            .done(items => {
              this.setState({ items, error: null })
            })
            .error(error => {
              this.setState({ items: [], error })
            })
        }

Theses

  • reactive, strict, explicit dataflow - state & props (parent state) to be the only sources of truth for the render
  • much like Angular 1, but better because allows component composition (Angular 1 controller composition and data exchange is hell)
  • complex async flow is still painful (nothing changed)

Reactive, Stateful, Pure

(2015-)

Get to know "side-effects"

  • should be separated to keep functions & dataflow pure
  • pure functions & dataflow are much easier to test
  • main examples in web frontend are: backend api requests (AJAX), third-party api requests through iframe (e.g., OAuth SDKs)
  • best known in web frontend as part of Redux (FLUX) -like dataflow/architecture
  • main implementations in web frontend are: Redux-Thunk, Redux-Saga, Redux-Observable, NgRx

Get to know "side-effects"

Get to know Rx

  • focuses on onservable streams concept
  • includes more than 60 operators for stream manipulation
  • made for concurrency and asynchronicity, with reactive & functional programming principles in mind
  • available for JavaScript, Java, Scala, C#, C++, Clojure, Python and many others
  • is an important dependency of Angular 2+ toolset

In modern React or Angular2+ application

  • actions (or changes, or events) are stream
  • data is stream
  • purity achieved as side-effects isolated
  • complex async flow is easier to implement as Rx introduced

          const { fetchItems, items, error } = this.props

        }

The difference?

  • entities in both examples are the same (state, actions, reducers, selectors, subscriptions)
  • dataflow for both platforms is the same
  • side-effect model (and code!) for both platforms is the same
  • main difference is in rendering and data binding (React JSX & VirtualDOM vs Angular2+ templates & ShadowDOM, React props vs Angular2+ Input/Output decorators, etc)
  • there are mostly syntactic (and implementation) differences between platforms

Rx benefits

  • Rx allows for intermediate results of a complex async flow
  • Rx allows for cancellation of an async flow
  • Rx allows for very complex stream manipulations

Example: Rx complex async flow 

        function fetchItemsEffect (action$) {
          const dispatchPending = () => of({ type: '@@fetchItems/pending' })
          const dispatchStepTwo = () => of({ type: '@@fetchItems/stepTwo' })
          const fetchItems = () => ajax({ method: 'GET', url: '/items' })
          const fetchItemsMeta = items => ajax({ method: 'GET', url: '/items/meta', payload: items })
         
          return action$.pipe(
            ofType('@@fetchItems'),
            switchMap(() => merge(
              dispatchPending(),
              fetchItems().pipe(
                switchMap(items => merge(
                  dispatchStepTwo(),
                  fetchItemsMeta(items).pipe(
                    tap(payload => { console.log('successfully fetched items meta', payload) }),
                    switchMap(payload => fetchItemsSuccess(payload.response))
                  )
                ))
              )
            )),
            catchError(payload => fetchItemsError(payload.response)),
            takeUntil(action$.ofType('@@fetchItems/cancel'))
          )
        }

        function component (sources) {

            select('button'),
            events('click')
          )

            switchMap(() => ajax({ method: 'GET', url: '/items' })),

            map(payload => payload.response)
          )
          const vdom$ = fetchItemsResult$.pipe(
            map(result => (
              <div>

                { Array.isArray(result)

              </div>
            )

Theses

  • actions (or changes, or events) are stream
  • data is stream
  • DOM updates are stream
  • DOM events are stream
  • all front-end application logic is a stream manipulation logic

        const context$ = new Rx.Subject()
         
        function handleFetchItemsRequest (context) {
          const items = getItemsFromDb()
          context.response.writeHead(200, { 'Content-Type': 'application/json' })
          context.response.end(JSON.stringify(items))
        }
         
        context$
          .pipe(
            tap(context => { console.log('request from ip', context.request.connection.remoteAddress) })
          )
          .subscribe(handleFetchItemsRequest)
         
        const server = http.createServer((request, response) => {
          context$.next({ request, response })
        })
         
        server.listen(3000)
